T1201.000.02 도메인 비밀번호 정책 조회

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

  • CurrentProcess = "cmd.exe" AND
  • Action = "CreateProcess" AND
  • TargetProcess = "net.exe" AND
  • TargetCmdLine = "account"

    Detection(EDR)

Response

  • 공격자에 의해 PC가 제어되었을 가능성이 있으므로 호스트 네트워크를 격리합니다.
  • 노출된 계정의 권한을 변경합니다.
  • 노출된 계정의 비밀번호를 변경합니다.

Mitigations

  1. 네트워크 트래픽 모니터링 및 차단

    • 동적 데이터 확인을 수행하는 악성 코드가 C2 서버와 통신하지 못하도록 방화벽 및 네트워크 보안 정책을 설정합니다.
    • DNS 및 웹 트래픽 필터링을 사용하여 악성 도메인과의 통신을 차단합니다.

  2. 애플리케이션 제어(Application Control

    • 승인된 애플리케이션 및 스크립트만 실행되도록 제한하여 악성 실행 파일이 동적 데이터를 가져오는 것을 방지합니다.
    • Microsoft AppLocker 또는 Software Restriction Policies(SRP)를 활용할 수 있습니다.

  3. 메모리 분석 및 코드 실행 모니터링

    • 메모리 내에서 로드된 코드나 동적으로 해석되는 데이터를 탐지하기 위해 EDR(Endpoint Detection and Response) 솔루션을 사용합니다.
    • Process Hollowing 및 Reflective DLL Injection과 같은 동적 코드 로딩 기법을 감시합니다.

  4. 파일 무결성 모니터링(File Integrity Monitoring, FIM

    • 실행 파일 및 중요 시스템 파일이 변조되거나 새로운 데이터가 삽입되는지 정기적으로 검사합니다.

  5. 보안 정책 및 사용자 교육

    • 이메일 첨부 파일, 의심스러운 다운로드, 또는 매크로 실행과 관련한 보안 인식을 높여 동적 데이터 확인을 악용한 공격을 방지합니다.
