T1056.003.01 Input Capture - Web Portal Capture

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

Detection(EDR)

Response

  1. 계정 관련 대응: 노출된 계정의 비밀번호를 변경합니다.

  2. 네트워크 관련 대응: 공격자에 의해 PC가 제어되었을 가능성이 있으므로 호스트 네트워크를 격리합니다. 공격자 C&C 서버로 추정되는 주소를 차단합니다.

Mitigations

  1. 웹 애플리케이션 보안

    • 입력 검증 강화:
      • 웹 포털에서 사용자 입력(특히 URL, 양식 데이터)을 철저히 검증합니다.
    • 콘텐츠 보안 정책(CSP):
      • 웹 애플리케이션에서 외부 리소스의 실행을 제한하기 위해 CSP를 설정합니다.
    • 크로스 사이트 스크립팅(XSS) 방지:
      • XSS 방지를 위해 HTML과 JavaScript 출력 시 적절한 이스케이프 처리를 수행합니다.

  2. 사용자 계정 보안

    • 2단계 인증(2FA):
      • 계정 탈취를 방지하기 위해 2FA를 활성화합니다.
    • 비밀번호 관리:
      • 비밀번호가 강력하고 주기적으로 변경되도록 권장합니다.

  3. 브라우저 환경 보호

    • 확장 프로그램 관리:
      • 사용자가 브라우저 확장 프로그램을 설치하기 전에 신뢰할 수 있는 소스에서만 설치하도록 제한합니다.
    • 보안 설정 강화:
      • 브라우저의 보안 설정(예: 팝업 차단, 스크립트 제한)을 강화합니다.

  4. 네트워크 보안

    • SSL/TLS 사용:
      • 웹 포털에서 데이터 전송 시 SSL/TLS 암호화를 강제하여 중간자 공격을 방지합니다.
    • 웹 애플리케이션 방화벽(WAF):
      • 악성 요청을 차단하기 위해 WAF를 설정합니다.

  5. 교육 및 훈련

    • 사용자 교육:
      • 악성 링크를 피하고 의심스러운 브라우저 확장 프로그램 설치를 자제하도록 교육합니다.
    • 보안 팀 훈련:
      • 보안 팀이 최신 위협에 대응할 수 있도록 정기적인 훈련을 수행합니다.
