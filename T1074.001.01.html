Response

의심스러운 파일이나 디렉토리가 발견된 경우, 이를 즉시 격리하여 추가적인 손상을 방지하고, 분석 후 필요 시 삭제합니다.

Mitigations

  1. 로컬 파일 및 디렉터리 모니터링 (M1047 - System Logging & Monitoring)

    • 로컬 파일 시스템을 지속적으로 모니터링하고, 비정상적인 파일 생성이나 이동을 감지
    • 디렉터리 모니터링 도구(예: Sysmon)를 사용하여 불법적인 파일 저장 경로나 비정상적인 데이터 스테이징을 실시간으로 추적
    • 파일 생성 및 변경 활동을 기록하고, 의심스러운 파일 패턴을 탐지하여 경고

  2. 파일 무결성 모니터링 (M1049 - File Integrity Monitoring)

    • 파일 무결성 모니터링을 설정하여 중요한 시스템 파일이나 민감한 데이터의 변조를 방지
    • 파일 변경, 이동, 삭제를 추적하여 데이터 유출의 징후를 실시간으로 탐지하고 경고
    • 로컬 드라이브나 네트워크 공유 폴더에서 이상적인 데이터 접근 패턴을 설정하여 비정상적인 활동을 식별

  3. 정기적인 시스템 감사 및 점검 (M1038 - Execution Prevention)

    • 시스템 감사를 정기적으로 수행하여 로컬 시스템에 의심스러운 프로그램이나 데이터 스테이징 파일이 저장되지 않도록 점검
    • 보안 정책을 통해 불법적인 데이터 이동이나 비정상적인 파일 다운로드를 차단하고, 스테이징 파일의 생성을 막음
    • 파일 접근에 대한 권한 관리를 강화하여 불필요한 데이터 저장을 방지

  4. 데이터 암호화 및 전송 보안 강화 (M1029 - Data Backup)

    • 로컬 시스템에 저장되는 데이터를 암호화하여 파일 훼손이나 도난 방지
    • 민감한 데이터를 로컬 드라이브에 저장하기 전에 암호화하고, 보안 네트워크 전송을 위해 데이터 보호 시스템을 설정
    • 자동화된 백업 시스템을 구축하여 데이터 손실이나 유출을 방지하고, 악성 파일의 전송을 추적

  5. 네트워크 보안 강화 및 세분화 (M1037 - Network Segmentation)

    • 내부 네트워크와 외부 네트워크를 세분화하여 로컬 데이터 스테이징 후 외부 전송을 차단
    • 네트워크 세분화를 통해 의심스러운 데이터 전송을 차단하고, 비정상적인 데이터 이동을 모니터링
    • 내부 네트워크에 대한 접근을 제한하여 데이터 스테이징을 막고, 외부와의 연결을 통제하는 네트워크 방어 강화

  6. 정책 기반 데이터 전송 및 관리 (M1035 - Access Control)

    • 파일 접근 및 이동에 대한 엄격한 정책을 설정하여 중요 파일이나 데이터를 비인가된 위치에 스테이징하지 못하도록 방지
    • 파일 이동 및 파일 저장 규정을 설정하여 공격자가 로컬 시스템에 데이터를 저장하지 못하도록 제한
    • 데이터 이동에 대한 정책을 강화하여 보안을 강화하고, 비정상적인 파일 이동을 모니터링

  7. 실시간 악성 코드 탐지 및 예방 (M1030 - Malware Protection)

    • 실시간 악성 코드 탐지 시스템을 설정하여 로컬 데이터 스테이징을 통한 악성 파일 전파를 차단
    • 엔드포인트 보호 시스템을 사용하여 로컬 시스템에서 악성 파일이나 스테이징된 데이터를 탐지하고 삭제
    • 파일 스캔 및 디지털 서명을 사용하여 악성 파일이 로컬 시스템에 저장되지 않도록 방지

  8. 사용자 교육 및 보안 인식 강화 (M1019 - Awareness and Training)

    • 사용자 교육을 통해 데이터 스테이징의 위험성과 보안 정책을 인식시키고, 불필요한 데이터 저장을 방지
    • 보안 인식 교육을 통해 직원들이 로컬 드라이브에 불필요한 데이터 저장을 하지 않도록 유도
    • 파일 저장 및 전송 규정을 강화하여 직원들이 보안 규정을 준수하도록 교육
