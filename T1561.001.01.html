T1561.001.01 디스크 콘텐츠 파괴

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

탐지 방법 작성, EFR과 무관하게 해당 Action에 대한 탐지 방법 명시

Detection(EDR)

EDR에 관련된 대응 방안을 작성

Response

디스크나 파일 시스템에서 데이터를 삭제하는 작업을 감지하기 위해, 시스템 로그를 분석 하고 데이터 백업을 자주 수행하여 백업이 안전한 위치에 저장되도록 관리합니다.

Mitigations

1. 정기적인 데이터 백업 및 복구 전략 수립

  • 3-2-1 백업 전략을 적용하여 데이터를 보호합니다.
    • 3개의 데이터 복사본을 유지합니다.
    • 2개의 서로 다른 저장 매체에 저장합니다.
    • 1개의 백업을 오프라인 또는 변경 불가능(Immutable)하게 저장합니다.
  • 백업 데이터를 네트워크에서 격리(Isolation)하여 공격자가 접근할 수 없도록 보호합니다.
  • 백업 복구 테스트를 정기적으로 수행하여 공격 발생 시 신속한 복구가 가능하도록 합니다.

2. 최소 권한 원칙(Least Privilege) 적용

  • 일반 사용자에게 디스크 삭제 권한을 제한하여 공격자가 관리자 또는 시스템 권한을 악용하지 못하도록 합니다.
  • 중요한 시스템 파일 및 디스크 관련 명령어(diskpart, dd, wipe, format 등)의 실행을 관리자 권한으로 제한합니다.

3. 시스템 및 파일 무결성 모니터링

  • File Integrity Monitoring(FIM)을 적용하여 디스크 삭제 시도를 감지합니다.
  • SIEM(Security Information and Event Management) 솔루션을 활용하여 비정상적인 디스크 접근 및 삭제 시도를 실시간 감지하고 경고를 생성합니다.
  • Windows 이벤트 로그 및 Sysmon을 설정하여 디스크 삭제 명령어 실행 여부를 추적합니다.

4. 엔드포인트 보호 및 이상 행위 탐지

  • EDR(Endpoint Detection and Response) 및 NGAV(Next-Generation Antivirus) 솔루션을 활용하여 비정상적인 디스크 삭제 활동을 탐지하고 차단합니다.
  • Windows Defender Attack Surface Reduction(ASR) 및 보안 정책을 적용하여 악성코드가 디스크 삭제 작업을 수행하지 못하도록 제한합니다.
  • SIEM 및 SOAR(Security Orchestration, Automation, and Response) 솔루션을 활용하여 자동 대응을 설정합니다.

5. 보안 업데이트 및 취약점 관리

  • 운영체제 및 보안 소프트웨어를 최신 상태로 유지하여 디스크 삭제를 유발하는 취약점(예: CVE-2021-40444 같은 Windows 취약점)으로부터 보호합니다.
  • 정기적인 취약점 점검(Vulnerability Scanning)을 수행하여 공격자가 악용할 수 있는 보안 허점을 사전에 차단합니다.

6. 사용자 교육 및 정책 적용

  • 내부 직원 보안 교육을 통해 의심스러운 실행 파일이나 디스크 삭제 명령어 실행을 방지하도록 유도합니다.
  • 디스크 삭제 방지 그룹 정책(GPO) 적용하여 비인가 사용자가 디스크 삭제 명령어를 실행할 수 없도록 제한합니다.
