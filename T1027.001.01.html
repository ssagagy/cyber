T1027.001.01 난독화된 파일 또는 정보:바이너리 패딩

Response

  1. 파일 분석: 변조된 파일이 의심되는 경우, 샌드박스 환경이나 격리된 시스템에서 해당 파일을 분석합니다.
  2. 의심 파일 격리: 변조된 파일이 발견된 경우, 즉시 해당 파일을 격리하여 추가적인 실행이나 확산을 방지합니다. 이를 통해 악성 파일이 시스템에서 동작하지 않도록 해야 합니다.
  3. 파일 삭제: 분석 결과 악성 파일로 확인된 경우, 시스템에서 해당 파일을 삭제하거나 격리된 저장소로 이동시켜 시스템 내에서의 위험을 제거합니다.
  4. 네트워크 격리: 변조된 파일이 시스템 내에 존재하여 악성 동작을 수행하는 경우, 해당 시스템 또는 네트워크를 격리하여 다른 시스템으로 확산되는 것을 방지합니다.
  5. 시스템 로그 및 보안 이벤트 분석: 변조된 파일이 생성되거나 실행된 시점에 대한 로그를 분석하여 어떤 행위가 발생했는지 추적합니다. 공격자의 활동을 조사하여 감염 경로 및 추가적인 감염 여부를 확인합니다.

Mitigations

  1. 파일 특성 기반 탐지
    • 파일의 해시 값이 변경되더라도 탐지가 가능하도록 파일의 다른 특성(파일 크기, 메타데이터, 서명 정보, 코드 패턴 등)을 분석하는 보안 솔루션을 사용합니다. 해시 값만을 이용한 탐지는 우회될 수 있기 때문에 파일의 구조나 동작 방식 등 다양한 기준을 활용한 탐지 방식이 필요합니다.
  2. 보안 솔루션의 최신화
    • 안티바이러스 및 EDR 솔루션을 최신 상태로 유지하여 패딩을 추가해 해시를 변경한 변종 악성코드도 탐지할 수 있도록 보안 제품을 정기적으로 업데이트합니다. 새로운 변종에 대한 탐지 패턴을 지속적으로 적용하는 것이 중요합니다.
  3. 변경 이력 관리
    • 시스템 내 중요한 파일의 무결성을 확인하기 위해 정기적으로 파일 변경 이력을 관리하고, 무결성 검사를 수행합니다.
  4. 파일 공유 및 다운로드 경로 차단
    • 알려진 악성코드가 다운로드되는 경로를 추적하고, 해당 경로를 네트워크에서 차단하여 변조된 파일이 내부 시스템으로 유입되지 않도록 방지합니다.
  5. 파일 패딩 추가 탐지
    • 특정 파일이 패딩을 추가하여 크기를 비정상적으로 증가시키는 경우 이를 탐지할 수 있는 규칙을 설정합니다. 비정상적으로 파일 크기가 증가하는 행위는 해시 변조를 의심할 수 있는 신호가 될 수 있습니다.
