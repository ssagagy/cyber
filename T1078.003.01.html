T1078.003.01 유효한 계정: 로컬 계정

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

로컬 계정 악용을 탐지하려면 비정상적인 계정 생성 또는 로그인 활동을 모니터링해야 한다.

Detection(EDR)

Response

  1. 계정 비활성화 및 암호 변경 : 침해된 로컬 계정이 확인되면 즉시 계정을 비활성화하거나 암호를 변경해야 한다. 관리 계정이나 중요 계정이 공격당한 경우 시스템 전체의 암호 정책을 재검토하고, 다중 인증(MFA)을 적용하는 것을 권고한다.
  2. 악성 활동 조사 : 공격자가 로컬 계정을 사용해 어떤 작업을 수행했는지 조사한다. 특히 권한 상승, 파일 접근, 네트워크 액세스, 악성 코드 실행 등을 분석해야 한다. 이벤트 로그를 활용해 침해 범위를 파악하고, 추가적인 피해가 발생했는지 확인한다.
  3. 격리 및 포렌식 조사 : 침해된 시스템을 네트워크에서 격리하고, 포렌식 분석을 통해 악성 활동을 조사한다. 공격자의 접근 경로와 탈취 방법을 분석하고, 이를 차단할 방안을 모색한다.
  4. 시스템 복구 및 패치 : 공격에 사용된 취약점을 제거하고 시스템을 패치한다. 시스템 복구 후 강력한 보안 설정을 적용해 추가 공격을 방지한다.

Mitigations

  1. 강력한 암호 정책

    • 로컬 계정의 암호를 강력하게 설정하고, 정기적으로 암호를 변경하는 정책을 적용한다.
    • 암호 복잡성, 길이, 만료 기간 등을 강화하고, 동일한 암호가 여러 시스템에서 사용되지 않도록 한다.

  2. 계정 권한 최소화

    • 로컬 계정에 불필요한 권한을 부여하지 않으며, 최소 권한 원칙(Least Privilege Principle)을 적용한다.
    • 특히 관리자 권한을 가진 로컬 계정이 불필요하게 사용되지 않도록 제한한다.

  3. 다중 인증 (MFA) 적용

    • 로컬 계정에 대해 다중 인증(MFA)을 적용해 계정이 탈취되더라도 추가적인 인증 절차를 통해 공격자의 접근을 차단한다.

  4. 사용하지 않는 계정 비활성화

    • 사용하지 않는 로컬 계정을 주기적으로 점검하고, 비활성화하거나 제거한다.
    • 기본적으로 생성된 관리자 계정도 필요하지 않으면 비활성화한다.

  5. 로그 모니터링 및 알림 설정

    • 로그를 실시간으로 모니터링하고, 비정상적인 계정 활동에 대해 경고를 설정해 조기에 대응할 수 있도록 한다.
