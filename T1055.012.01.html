T1055.012.01 Process Hollowing

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

메모리 내에서 실행 중인 프로세스의 상태 변화 및 비정상적인 메모리 사용 패턴과 공격자가 주로 사용하는 API 패턴을 모니터링 합니다.

Detection(EDR)

Response

  • 이상 징후 모니터링: 프로세스 행동, 네트워크 트래픽, 시스템 로그를 지속적으로 모니터링합니다.
  • 빠른 격리: 감염 의심 시스템을 신속하게 네트워크에서 격리합니다.
  • 포렌식 분석: 감염된 시스템의 메모리와 디스크를 분석하여 공격 경로와 영향을 파악합니다.
  • IoC(Indicators of Compromise) 업데이트: 발견된 악성 지표를 보안 시스템에 적용하여 추가 감염을 방지합니다.
  • 패치 및 업데이트: 취약점을 악용한 공격이었다면, 관련 시스템의 보안 패치를 즉시 적용합니다.

Mitigations

  1. 애플리케이션 격리

    • 샌드박스 기술을 사용하여 애플리케이션을 격리합니다.

  2. 행위 기반 탐지

    • 이상 행동을 감지하는 엔드포인트 보호 솔루션을 도입합니다.

  3. 권한 최소화

    • 사용자와 프로세스의 권한을 필요한 최소한으로 제한합니다.

  4. 코드 서명

    • 신뢰할 수 있는 소프트웨어만 실행되도록 코드 서명을 적용합니다.

  5. 메모리 보호

    • DEP(Data Execution Prevention)와 ASLR(Address Space Layout Randomization) 등의 메모리 보호 기술을 활성화합니다.
