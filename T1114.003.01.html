T1114.003.01 공격 대상 메일 규칙 및 정보 수집

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

  • 행동 유형 : AMSI_SCAN
  • 오퍼레이션 : contains
  • 값1 : New-PSSession
  • 값2 : microsoft.exchange
  • 값3 : New-InboxRule
  • 값4 : -Mailbox
  • 값5 : -MoveToFolder

Detection(EDR)

https://172.18.10.125:8903/#/analysis/incident/67a99a90002dc6ce0000189c

Response

  • 의심스러운 계정 즉시 비활성화 및 비밀번호 초기화
  • 공격자가 사용한 IP, 기기 정보 차단
  • 유출된 이메일 내용이 있는지 포렌식 분석 수행
  • 추가 피해를 방지하기 위해 이메일 서버 및 클라이언트 업데이트 적용

Mitigations

  1. 원격 이메일 프로토콜 보안 강화

  • IMAP, POP3, SMTP 인증 보안 강화 (OAuth2, TLS 적용)

  • 기본 인증(Basic Authentication) 비활성화하고 모던 인증(Modern Authentication, OAuth2) 적용

  • Exchange Web Services (EWS), Outlook Web Access (OWA) 등의 API 접근 제한

  1. MFA 및 계정 보안 강화

  • MFA (Multi-Factor Authentication) 필수 적용

  • 비밀번호 정책 강화 (최소 12자 이상, 특수문자 포함, 정기적 변경)

  • 해킹된 계정으로 이메일 접근을 방지하기 위해 조건부 접근 정책(Conditional Access Policy) 설정

  1. 네트워크 및 원격 접근 제한

  • VPN 및 특정 IP에서만 이메일 서버 접근 허용

  • Geo-IP 차단 설정하여 의심스러운 국가에서의 접근 제한

  • 이메일 서버 방화벽 규칙 적용하여 불필요한 원격 프로토콜 차단

  1. 이메일 보안 및 암호화 적용

  • 이메일 암호화 적용 (S/MIME, PGP)하여 이메일 데이터 보호

  • DLP(Data Loss Prevention) 솔루션 사용하여 이메일 데이터 유출 방지

  • 이메일 첨부 파일 및 링크에 대한 보안 검토 수행
