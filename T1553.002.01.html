T1553.002.01 인증서 서명

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

악성 파일이 서명되어 있는 경우 탐지합니다.

Detection(EDR)

Response

공격자가 생성한 파일을 삭제합니다.

Mitigations

  1. 인증 및 권한 부여 프로세스 강화 (M1028 - Application Layer Filtering)

    • 강력한 인증 방식(예: Multi-Factor Authentication, MFA)을 적용하여 인증 절차를 두 단계 이상으로 강화
    • 애플리케이션 계층에서 인증 우회 공격을 방지하기 위해 세션 관리 및 토큰 기반 인증을 활용하여 인증된 세션이 아닌 경우 접근을 차단
    • 애플리케이션에서 기존 인증 절차를 강화하고, 인증 우회가 불가능하도록 암호화된 통신을 요구

  2. 애플리케이션 계층 프로토콜 모니터링 (M1033 - Process Monitoring)

    • 애플리케이션 계층 프로토콜(HTTP, FTP 등)을 모니터링하여 비정상적인 요청이나 우회 시도를 탐지
    • 정기적인 로그 분석과 패턴 분석을 통해 애플리케이션에서 비정상적인 인증 요청이나 악의적인 요청을 실시간으로 탐지
    • 로그 분석 도구를 사용하여 의심스러운 트래픽 패턴을 확인하고, 이를 통해 인증 우회 시도에 대응

  3. 입력 검증 및 취약점 분석 (M1040 - Security Configurations)

    • 애플리케이션에서 입력 검증을 강화하여 악의적인 요청이나 패턴이 시스템에 전달되지 않도록 방지
    • 웹 애플리케이션 방화벽(WAF)을 사용하여 HTTP 프로토콜을 통한 악성 요청이나 인증 우회 시도를 차단
    • 애플리케이션 보안 취약점 분석을 통해 인증 우회와 관련된 취약점을 사전에 발견하고 해결
    • 취약점 스캐너와 정적 분석 도구(SAST)를 사용하여 인증 우회에 영향을 미칠 수 있는 보안 취약점을 사전 점검

  4. TLS/SSL 암호화 및 통신 보안 강화 (M1032 - Application Layer Filtering)

    • 애플리케이션 통신에 대한 암호화(TLS/SSL)를 적용하여 중간자 공격을 방지하고, 인증 정보를 안전하게 보호
    • HTTPS를 통해 암호화된 채널을 사용하여 애플리케이션 계층에서 전송되는 인증 정보를 보호하고, 인증 우회를 방지
    • SSL/TLS 인증서 검증 및 강력한 암호화 알고리즘을 적용하여 보안 강화를 위한 기준을 준수

  5. 애플리케이션 및 네트워크 보안 감사 (M1047 - System Logging & Monitoring)

    • 애플리케이션 보안 감사 및 네트워크 보안 감사를 수행하여 인증 관련 보안 로그를 주기적으로 점검
    • 세션 및 인증 로그를 통해 비정상적인 로그인 시도나 인증 우회 시도를 탐지하고 이를 실시간으로 차단
    • 네트워크 트래픽 분석을 통해 애플리케이션 계층에서 발생하는 악성 요청을 차단하고, 공격자 IP 차단 및 접근 제어를 강화
