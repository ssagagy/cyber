T1652.000.01 장치 드라이버 검색

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

탐지 방법 작성, EFR과 무관하게 해당 Action에 대한 탐지 방법 명시

Detection(EDR)

Response

  • 공격자가 생성한 파일을 삭제합니다.
  • 결과물 출력 파일을 삭제합니다.

  • 공격자에 의해 PC가 제어되었을 가능성이 있으므로 호스트 네트워크를 격리합니다.

    Mitigations

    1. 정기적인 데이터 백업 및 복구 전략 수립

      • 3-2-1 백업 전략(3개의 데이터 복사본을 2가지 다른 저장 매체에 보관하고, 1개는 오프사이트 보관)을 적용합니다.
      • 백업 데이터를 변경 불가능한 상태(Immutable Backup) 로 저장하여 공격자가 암호화하거나 삭제하지 못하도록 합니다.
      • 백업 복구 테스트를 정기적으로 수행하여 복원 가능성을 확인합니다.

    2. 최소 권한 원칙 적용 및 네트워크 격리

      • 사용자 및 서비스 계정에 최소 권한 원칙(Least Privilege) 을 적용하여 불필요한 쓰기 및 삭제 권한을 제한합니다.
      • 중요한 시스템 및 데이터 저장소를 네트워크에서 격리하여 랜섬웨어가 확산되지 않도록 합니다.

    3. 파일 무결성 모니터링 및 이상 탐지

      • File Integrity Monitoring(FIM) 도구를 사용하여 중요한 데이터 파일이 비정상적으로 변경되었는지 감지합니다.
      • 보안 솔루션(SIEM, EDR)을 활용하여 대량의 파일 암호화와 같은 비정상적인 파일 활동을 탐지하고 경고를 생성합니다.

    4. 랜섬웨어 방어 기능 적용

      • Windows Defender, CrowdStrike, SentinelOne 등의 EDR/XDR 솔루션을 사용하여 랜섬웨어 동작을 감지하고 차단합니다.
      • Windows의 Controlled Folder Access 기능을 활성화하여 중요한 폴더가 무단으로 변경되지 않도록 보호합니다.

    5. 사용자 교육 및 피싱 방어

      • 피싱 이메일 시뮬레이션 훈련을 통해 사용자가 악성 첨부파일이나 링크를 열지 않도록 교육합니다.
      • 기업 내 랜섬웨어 감염 사고 사례를 공유하고, 보안 인식을 높입니다.

    6. 네트워크 및 프로세스 감시

      • PowerShell 및 WMI 명령어를 통한 암호화 작업을 감지할 수 있도록 로깅을 활성화합니다.
      • 랜섬웨어가 C2(Command and Control) 서버와 통신하지 못하도록 DNS 필터링 및 방화벽 정책을 강화합니다.
