T1562.000.01 시스템 저널 로깅 중단

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

cmd line: systemctl AND stop AND systemd-journald

Detection(EDR)

Response

공격자가 보안 툴을 비활성화하거나 수정하여 시스템의 방어를 약화시키는 것을 방지하는 데 중점을 둡니다. 이를 효과적으로 대응하려면 보안 툴의 무력화 또는 변경을 탐지하고 차단하는 기술적 조치가 필요합니다.

Mitigations

  1. 보안 소프트웨어 및 시스템의 지속적인 모니터링 (M1033 - Process Monitoring)

    • 보안 소프트웨어(예: 안티바이러스, 침입 탐지 시스템)의 실시간 모니터링을 강화하고, 비정상적인 프로세스 활동이나 방어 메커니즘 비활성화 시도를 탐지
    • 시스템 로그를 모니터링하여 보안 관련 이벤트나 방어 시스템 비활성화가 발생할 경우 즉시 알림을 받도록 설정
    • 보안 소프트웨어의 자동 업데이트와 패치 관리를 통해 최신 위협에 대응하도록 한다

  2. 방어 시스템에 대한 액세스 제한 (M1045 - Restrict Access to Tools)

    • 보안 소프트웨어나 방어 시스템에 대한 접근 권한을 최소화하고, 불필요한 접근을 차단
    • 관리자 권한을 가진 사용자만 방어 시스템의 설정 변경이나 비활성화를 할 수 있도록 제한
    • 시스템 관리자 및 보안 팀 외에는 보안 소프트웨어나 보안 도구에 접근할 수 없도록 강력한 인증 및 권한 부여 절차를 적용

  3. 보안 설정 및 정책 강화 (M1032 - Application Layer Filtering)

    • 운영 체제 및 애플리케이션 보안 설정을 강화하여 방어 시스템에 대한 우회 시도를 방지
    • 보안 정책을 명확히 정의하고, 정기적인 보안 감사와 보안 설정 점검을 통해 방어 메커니즘이 적절히 작동하는지 확인
    • 애플리케이션 계층 보안을 강화하여 악성 코드나 방어 우회 시도를 차단

  4. 디지털 서명 및 무결성 검증 (M1040 - Security Configurations)

    • 파일 무결성 모니터링 및 디지털 서명을 활용하여 방어 시스템에 대한 변경이나 삭제가 발생하지 않도록 방지
    • 무결성 검사 도구를 사용하여 시스템에 설치된 보안 소프트웨어나 방어 체계의 상태를 정기적으로 점검
    • 무단 변경이 발견되면 자동으로 경고 및 알림을 발생시켜 빠르게 대응할 수 있도록 설정

  5. 보안 경고 및 이벤트 응답 (M1031 - Network Segmentation)

    • 보안 경고 시스템을 구축하여 방어 시스템 비활성화 시도나 비정상적인 활동에 대해 즉시 대응할 수 있도록 준비
    • 침입 탐지 시스템(IDS) 및 침입 방지 시스템(IPS)을 활성화하여 네트워크 트래픽에서 발생할 수 있는 공격 패턴을 모니터링하고 실시간으로 차단
    • 침해 사고 대응 계획을 수립하여 방어 메커니즘 손상을 포함한 보안 사고 발생 시 즉시 대응할 수 있도록 훈련

  6. 애플리케이션 및 시스템 패치 관리 (M1047 - System Logging & Monitoring)

    • 시스템과 애플리케이션에 대한 정기적인 패치 관리를 통해 보안 취약점을 해결하고 방어 시스템을 최신 상태로 유지
    • 패치 및 업데이트 자동화 시스템을 사용하여 보안 취약점이 발생하지 않도록 방지
    • 패치 로그 및 업데이트 기록을 보관하여 취약점 해결 여부를 추적하고, 비활성화된 방어 시스템을 재설정
