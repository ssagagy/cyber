T1562.002.01 Disable Windows Event Logging

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

탐지 방법 작성, EFR과 무관하게 해당 Action에 대한 탐지 방법 명시

Detection(EDR)

EDR에 관련된 대응 방안을 작성

Response

공격자에 의해 PC가 제어되었을 가능성이 있으므로 호스트 네트워크를 격리합니다.

Mitigations

  1. 보안 소프트웨어의 무결성 및 보호 설정 (M1040 - Security Configurations)

    • 보안 도구(예: 안티바이러스, 침입 탐지 시스템)의 무결성을 보장하고, 비활성화나 중지를 차단하는 보호 기능을 활성화
    • 보안 소프트웨어가 자동으로 재시작되도록 설정하거나 핸즈프리로 복구할 수 있도록 구성하여, 도구가 비활성화되는 것을 방지
    • 파일 및 레지스트리 보호를 통해 보안 도구의 설정이 수정되거나 삭제되지 않도록 보호

  2. 관리자 권한 및 접근 제어 (M1045 - Restrict Access to Tools)

    • 관리자 권한을 최소화하고, 불필요한 사용자나 어플리케이션이 보안 도구에 접근할 수 없도록 액세스 제어를 강화
    • 보안 도구에 대한 변경 권한을 승인된 사용자로 제한하고, 강력한 인증 및 다단계 인증(MFA)을 요구하여, 무단 변경 시도를 차단
    • 권한 기반 접근 제어(PBAC)를 사용하여 보안 소프트웨어에 대한 불필요한 수정이나 비활성화를 방지

  3. 보안 도구에 대한 모니터링 및 경고 시스템 강화 (M1033 - Process Monitoring)

    • 보안 도구의 상태를 실시간으로 모니터링하고, 비정상적인 활동(예: 보안 소프트웨어 비활성화 시도)을 감지하는 모니터링 시스템 설정
    • 보안 도구가 비활성화되었을 경우, 자동 경고 시스템을 활성화하여 빠르게 대응할 수 있도록 알림을 발생시키고, 비정상적인 행동을 즉시 기록하여 추적할 수 있도록 설정
    • 시스템 로그와 보안 이벤트를 실시간으로 분석하여, 보안 도구의 비활성화 시도를 탐지하고 대응

  4. 소프트웨어 및 보안 도구 업데이트 자동화 (M1047 - System Logging & Monitoring)

    • 보안 도구가 최신 상태로 유지되도록 자동 업데이트를 설정하고, 패치 관리 시스템을 통해 최신 보안 패치가 적용되도록 한다
    • 업데이트 로그와 보안 패치 기록을 관리하여, 보안 도구가 최신 상태로 유지되고, 보안 취약점이 해결되도록 한다
    • 패치 관리 도구를 통해 정기적인 업데이트와 보안 패치 적용을 자동화하여 비활성화된 도구를 복구하고 취약점을 예방

  5. 디지털 서명 및 파일 무결성 검증 (M1055 - Secure Data Transmission)

    • 디지털 서명 및 무결성 검증을 통해 보안 도구나 보안 시스템의 파일 및 구성 요소에 대한 변경 여부를 검토
    • 파일 무결성 모니터링을 통해 보안 도구의 설정이 변경되었는지 확인하고, 무단으로 변경된 파일을 자동으로 복구
    • 파일 무결성 검사와 디지털 서명 검증을 통해 보안 도구의 코드가 변조되었는지 점검

  6. 보안 사고 대응 계획 (M1031 - Network Segmentation)

    • 보안 도구 비활성화 시도에 대한 대응 절차를 사전에 정의하여 침해 사고 대응 계획을 마련하고, 보안 도구 비활성화 시 즉시 대응할 수 있도록 준비
    • 침해 사고 발생 시 방어 기능이 손상된 상태에서의 복구 절차를 명확히 정의하고, 모의 훈련을 통해 신속하게 대응할 수 있도록 한다
    • 침입 탐지 시스템(IDS)와 침입 방지 시스템(IPS)을 사용하여 네트워크 트래픽에서 비정상적인 패턴을 탐지하고 실시간으로 차단
