Response

  • 노출된 계정의 권한을 변경합니다.
  • 노출된 계정의 비밀번호를 변경합니다.

    Mitigations

  1. 비밀번호 정책 조회 권한 제한 (M1026 - Privileged Account Management)

    • 일반 사용자가 비밀번호 정책을 조회할 수 없도록 권한 제한
    • Active Directory 환경에서 Get-ADDefaultDomainPasswordPolicy 실행을 관리자 계정으로만 허용
    • 로컬 정책 조회(net accounts) 실행을 관리자 권한으로 제한

  2. 보안 정책 강화 (M1027 - Password Policies)

    • 강력한 비밀번호 정책 설정 (최소 12~15자, 대소문자/숫자/특수문자 포함)
    • 비밀번호 재사용 제한 및 주기적인 변경 요구
    • 계정 잠금 정책 적용 (연속된 실패 시 계정 잠금)

  3. 로그 및 이벤트 모니터링 (M1047 - System Logging & Monitoring)

    • Windows 이벤트 로그 감시
      • Event ID 4738 (사용자 계정 속성 변경)
      • Event ID 4781 (사용자 비밀번호 변경 시도)
    • SIEM 솔루션 활용하여 비정상적인 정책 조회 탐지

  4. 비밀번호 정책 정보 보호 (M1032 - Multi-Factor Authentication & M1035 - Limit Access to Resource Over Network)

    • 비밀번호 정책이 노출되지 않도록 GPO 정책 보호
    • 비밀번호 관련 설정 파일(/etc/login.defs, /etc/security/pwquality.conf)의 접근 권한 제한

  5. 사용자 교육 및 인식 강화 (M1019 - Awareness and Training)

    • 비밀번호 정책 탐색이 보안 위험을 초래할 수 있음을 관리자 및 사용자에게 교육
    • 비밀번호 재사용 방지 및 MFA(다중 인증) 적용 교육 실시
