T1010.000.01 윈도우 프로세스 목록 정보 수집(GetProcesses)

D3FEND

MITRE ATT&CK 액션을 기준으로 대응 방안을 작성

Detection

None

Detection(EDR)

None

Response

  1. 비인가 프로세스 식별 탐지된 프로세스 목록 수집 행위의 주체를 확인합니다.

  2. 비인가 프로세스 종로 및 격리 의심스러운 프로세스를 즉시 종료하고, 실행 파일을 격리하여 추가적인 실행을 방지합니다.

  3. 분석 메모리, 로그, 침투 경로 등을 분석합니다.

Mitigations

  1. 권한 관리 강화 일반 사용자 계정에 프로세스 목록에 대한 접근 권한을 제한하고, 관리자 계정만 접근할 수 있도록 설정합니다. 최소 권한 원칙(Principle of Least Privilege)을 적용하여 불필요한 권한 상승을 방지합니다.

  2. 애플리케이션 화이트리스트 적용 신뢰할 수 있는 애플리케이션만 실행될 수 있도록 화이트리스트 정책을 적용합니다. Microsoft의 AppLocker나 Windows Defender Application Control (WDAC)을 활용해 허가되지 않은 프로세스 실행을 차단합니다.

  3. 안티바이러스 및 EDR 솔루션 배포 최신 버전의 안티바이러스 및 EDR(Endpoint Detection and Response) 솔루션을 배포하여 악성 코드 탐지 및 차단을 자동화합니다. 행동 기반 탐지를 활성화하여 비정상적인 프로세스 활동을 탐지합니다.

  4. 이벤트 로깅 및 감사 Windows 이벤트 로그에서 프로세스 생성 이벤트(이벤트 ID 4688)와 접근 시도 이벤트를 모니터링합니다. Windows Management Instrumentation (WMI) 접근 시도를 감시하여 의심스러운 활동을 탐지합니다.

  5. 네트워크 세그멘테이션 및 방화벽 설정 중요한 서버나 민감한 시스템은 네트워크 세그멘테이션을 통해 접근을 제한합니다. 외부 IP 주소로의 비정상적인 트래픽을 방화벽 규칙으로 차단합니다.
